## Codebase Patterns
- New unit test files under tests/unit/ MUST declare `pytestmark = pytest.mark.unit`
  (pyproject addopts = `-m unit`); without it the file is silently deselected (0 run).
- pytest defaults to unit only. Mark integration/e2e with `pytest.mark.integration`/`e2e`
  and skip-guard on the live resource (~/.sema/poc.duckdb, Neo4j, Databricks creds).
- mypy is strict (pyproject [tool.mypy] strict=true). Type hints everywhere; run
  `uv run mypy src/sema/` before committing.
- R29 boundary: src/sema/engine/ (and the generic spine) must contain NO OMOP/OncoTree
  literals (OncoTree/oncotree/cBioPortal/cbio, "Maps to", standard_concept,
  condition_occurrence, concept_id). Domain-specific vocab knowledge belongs OUTSIDE

  authored manifests (targets/manifests/*.yaml) and policy modules. The migration check
  lives in tests/unit/test_engine_coupling.py (US-001 will formalize the full guard).
- The targets/ framework (PR #109, ea2aed7) is MERGED to main and R29-clean. Build on it
  (load_target, ManifestTargetAdapter, materializer -> Neo4j/InMemory, `sema target load`);
  do NOT reimplement loaders/materializers. Do NOT grow/refactor loader.py or
  materializer_utils.py.
- Coverage gate >= 85% (`uv run pytest --cov=sema --cov-report=term-missing`); repo
  currently ~93%.
- ONCOTREE_CODE source-of-truth: `cbioportal_<study>.sample.ONCOTREE_CODE` only (2
  studies in ~/.sema/poc.duckdb). Discover studies via information_schema.columns
  (column_name='ONCOTREE_CODE' AND table_name='sample'); 64 distinct codes.
- §1.5(f) metric math is FROZEN in src/sema/eval/mapping_goldset_utils.py (US-002):
  derive_zone/classify_cell/ConfusionMatrix. US-012 MUST reuse it (no re-derived
  math). Recall denominator includes Zone-2 recall_miss (reduces to the doc's
  TP/(TP+WRONG+FN) when no Zone-2 predictions exist).

  is an EXTERNAL ORACLE (human/crosswalk), never Sema's resolver. Codes awaiting the

  - ONCOTREE_CODE lives ONLY in `cbioportal_<study>.sample.ONCOTREE_CODE` (2

    column_name='ONCOTREE_CODE' AND table_name='sample'.

    re-derive it.

    FP_map) and is reported SEPARATELY — never folded into the mapped_* metrics.
  - GoldLabel.UNLABELLED is a third sentinel (beyond the §1.5 RESOLVED|NO_MAP)
    representing "scaffolded, awaiting the human oracle". score() excludes these
    and lists them in GoldSetReport.unscored_unlabelled so they can never be
    silently scored as labelled.
