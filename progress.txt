## Codebase Patterns
- Assertion construction has ~9 call sites across 6 engine files, each via a local
  `_make_assertion` helper. To touch ALL assertions at once, use a Pydantic
  `@model_validator(mode="after")` on `Assertion` (src/sema/models/assertions.py) —
  it is the single choke point. Do NOT edit each call site.
- Name/alias text for HAS_ENTITY_NAME / HAS_PROPERTY_NAME / HAS_ALIAS / HAS_SYNONYM
  lives in `payload["value"]`. Entity/Property/Term MERGE keys in
  loader_utils.batch_upsert_* are built from these values, so anything affecting the
  value affects graph node identity.
- `loader._run(query, rows=rows)` — UNWIND-style batch calls pass `rows` as a KWARG,
  not positional. Mock side_effects must accept `rows=` to capture batch payloads.
- Pure helpers that the models layer must import (e.g. normalize_utils) must have NO
  sema imports, to avoid an engine->models import cycle. engine/__init__.py is empty
  so `from sema.engine.normalize_utils import ...` is cycle-safe.
- Quality gate for THIS repo: `uv run pytest` + `uv run mypy src/sema/`. Coverage
  baseline ~90%. ruff has ~64 pre-existing errors in untouched files — keep YOUR
  files ruff-clean but the broad `ruff check src/sema/` is not green on this branch.
- test_code_standards.py enforces <=400-line files / <=60-line funcs via a frozen
  allowlist+ratchet; new files must comply or the suite fails.
