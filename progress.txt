## Codebase Patterns
- NEW unit test files MUST set `pytestmark = pytest.mark.unit` (pyproject addopts is
  `-m unit`). Without it the file is silently DESELECTED — `uv run pytest <file>` shows
  "N deselected / 0 selected" and the tests never run (and coverage of the code under
  test looks 0%). All tests/unit/*.py peers declare it after imports.
- Per-table vs once-per-build phases: `materialize_unified` (src/sema/graph/materializer.py)
  runs PER TABLE inside process_table AND in _try_resume. Anything that must reason over
  ALL tables (e.g. vocabulary deprecation) must NOT live there — carry per-table facts on
  TableResult (src/sema/pipeline/build.py) and run the global step once in
  orchestrate.run_build after _collect_results. TableResult.active_vocabularies is the
  example (US-004); _try_resume must also populate such fields or resumed tables drop out
  of the union.
- Assertion construction has ~9 call sites across 6 engine files, each via a local
  `_make_assertion` helper. To touch ALL assertions at once, use a Pydantic
  `@model_validator(mode="after")` on `Assertion` (src/sema/models/assertions.py) —
  it is the single choke point. Do NOT edit each call site.
- Name/alias text for HAS_ENTITY_NAME / HAS_PROPERTY_NAME / HAS_ALIAS / HAS_SYNONYM
  lives in `payload["value"]`. Entity/Property/Term MERGE keys in
  loader_utils.batch_upsert_* are built from these values, so anything affecting the
  value affects graph node identity.
- `loader._run(query, rows=rows)` — UNWIND-style batch calls pass `rows` as a KWARG,
  not positional. Mock side_effects must accept `rows=` to capture batch payloads.
- Pure helpers that the models layer must import (e.g. normalize_utils) must have NO
  sema imports, to avoid an engine->models import cycle. engine/__init__.py is empty
  so `from sema.engine.normalize_utils import ...` is cycle-safe.
- Quality gate for THIS repo: `uv run pytest` + `uv run mypy src/sema/`. Coverage
  baseline ~90%. ruff has ~64 pre-existing errors in untouched files — keep YOUR
  files ruff-clean but the broad `ruff check src/sema/` is not green on this branch.
- test_code_standards.py enforces <=400-line files / <=60-line funcs via a frozen
  allowlist+ratchet; new files must comply or the suite fails. (As of US-002 the .py
  is gone — only a stale .pyc remains — so the ratchet is NOT currently enforced, but
  still honor <=400/<=60 and never grow loader.py(661)/materializer_utils.py(440).)
- Ref strings are NOT consistent across layers: the connector emits column refs
  as `{fqn}/{column}` (SLASH, databricks_utils.py), but L2/stage_utils emits
  `{table_ref}.{column}` (DOT). NEVER compare raw ref strings for node identity —
  parse with CanonicalRef.parse (src/sema/models/physical_key.py), which handles
  both forms, then match on the physical merge key. Physical node MERGE keys:
  :Table {name, schema_name, catalog}; :Column {name, table_name, schema_name,
  catalog} (NO datasource_id on the node despite NodeKey.* including it).
- Provenance assertion subject_ref/object_ref always point at a physical
  Table/Column (table_ref for has_entity_name/entity has_alias; col_ref for
  property/semantic_type/decoded/vocab/parent_of/join). subject_id/object_id on
  Assertion are vestigial — always None, never populate them; resolve via
  subject_ref instead (US-006).
- :Term identity is the composite key {vocabulary_name, code} (US-002). The namespace
  is NEVER null: vocabulary-less terms use UNSCOPED_VOCAB='_unscoped'
  (src/sema/graph/term_identity_utils.py: term_namespace()). A column's decoded terms
  are namespaced by their MATCHED vocabulary if a VOCABULARY_MATCH exists, else by the
  value-set name `{table}_{column}_values` (src/sema/graph/term_vocab_utils.py). Any
  NEW site that MERGEs/MATCHes a :Term MUST supply the same namespace the term was
  created with, or the MATCH silently misses (e.g. IN_VOCABULARY/MEMBER_OF edges).
